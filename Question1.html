HTML Entities

An HTML entity is a piece of text ("string") that begins with an ampersand ( & ) and ends with a semicolon ( ; ). HTML entities are frequently used to display reserved characters (which would otherwise be interpreted as HTML code), and invisible characters (like non-breaking spaces)

5 commonly used entities

& & Interpreted as the beginning of an entity or character reference.

< < Interpreted as the beginning of a tag

> > Interpreted as the ending of a tag

" " Interpreted as the beginning and end of an attribute's value.

© © Interpreted as the copyright sign.

Question 3. Define accessibility in the context of web development?

Accessibility in Web Development: Accessibility in web development refers to designing and coding websites in a way that ensures people with disabilities can perceive, understand, navigate, and interact with digital content effectively. It involves making web content inclusive and usable for everyone, regardless of their abilities.

Importance of Accessible Websites:

  1. Reading captions for a video when in noisy or quiet environments
  2. Adjusting screen brightness in a dark or bright room
  3. Reading a transcript of a presentation to reinforce the information learned Reading a transcript of a presentation to reinforce the information learned
  4. Using the keyboard to navigate a website when the mouse is not working

Three ways in which accessibility in html helps us

  1. Alternative Text : The alt attribute provides an alternate text for an image, if the user for some reason cannot view it (because of slow connection, an error in the src attribute, or if the user uses a screen reader).

  2. Declare the Language : You should always include the lang attribute inside the tag, to declare the language of the Web page. This is meant to assist search engines and browsers.
